Getting started with Docker and PHP

References: http://www.newmediacampaigns.com/blog/docker-for-php-developers

What is a Docker Container?

A container is a single service in a larger application.

For example, assume we are building a PHP application. We need:

  1. Web Server (e.g, nginx)
  2. Application Server (e.g, PHP-FPM)
  3. Database Server (e.g, MySQL)

Our application has three distinct services.

  • Each of these services can be separated into its own Docker container.
  • Three containers are linked together.

We have a complete application.

What is a Docker Image?

Docker instance like a Object and Docker image like a Class.

Just like a single PHP class can be used to instantiate many unique PHP objects, so too can Docker images to used to instantiate many unique Docker containers.

For example, we can reuse a single PHP-FPM Docker image to instantiate many unique PHP-FPM containers for each of our applications.

How to get Docker images?

  • Step 1: Find and use Docker images on Docker Hubs https://hub.docker.com/
  • Step 2: Build custom Docker images (if need).

Why re-invent the wheel if someone has already built a Docker image that solves our problem?
